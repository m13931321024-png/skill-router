子 Agent 协作与验收闭环机制

## 使用方式
`/orchestrate <任务描述>`

## 流程

### 1. 任务拆解 → 进度表
收到任务 "$ARGUMENTS" 后，先分析并输出进度表：

| # | 子任务 | 负责 Agent | 状态 | 验收标准 |
|---|--------|-----------|------|---------|
| 1 | ... | Agent-1 | ⬜ 待做 | ... |

### 2. 并行派发
- 将独立子任务分配给多个子 Agent 并行执行
- 每个子 Agent 收到明确的任务描述、输入、期望输出
- 子 Agent 类型选择：
  - `Explore` — 代码调研、文件搜索
  - `general-purpose` — 编码实现、测试
  - `Plan` — 方案设计

### 3. 验收闭环
每个子 Agent 完成后，主 Agent 执行验收：
- TypeScript 编译检查（`npx tsc --noEmit`）
- 检查文件是否正确创建/修改
- 确认接口类型定义与文档一致
- 更新进度表状态

### 4. 自动提交
全部验收通过后，调用 `/smart-commit` 进行分批本地提交。

### 5. 最终报告
输出最终进度表（全部 ✅），附带：
- 变更文件清单
- commit 记录
- 发现的待处理问题（如有）
