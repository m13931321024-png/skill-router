代码修改完成后，按模块和功能细节分批提交（中文描述），并将发现的其他模块问题记录到 /docs/todos

## 流程

### 1. 变更分析
- 运行 `git status` 和 `git diff` 查看所有变更
- 按模块/功能将变更分组（如：接口层、组件层、配置层）
- 每组变更对应一次独立 commit

### 2. 分批提交规则
- **commit message 使用中文**，格式：`<类型>(<模块>): <描述>`
- 类型：`feat`=新功能, `fix`=修复, `refactor`=重构, `chore`=配置/工具
- 示例：`feat(api): 新增字段池和BizConfig接口服务`
- **只提交到本地，不 push**
- 每次 commit 只包含相关联的文件，不混合不同模块的改动

### 3. 问题记录
- 如果在修改过程中发现其他模块的问题（bug、技术债、待优化项）
- 记录到 `/docs/todos/` 目录下对应模块的文件中
- 格式：`- [ ] 问题描述 (发现时间: YYYY-MM-DD)`

### 4. 提交后验证
- 运行 `git log --oneline -10` 确认提交记录
- 运行 TypeScript 编译检查确认无新增错误

## 注意事项
- 不要使用 `git push`
- 不要使用 `--amend` 修改已有提交
- 不要使用 `git add .` 或 `git add -A`，逐个文件添加
